Instructions
  1. Fixes that are not bug fixes (e.g., documentation, style) should be labeled as False.
  2. Specifically, typos and spelling errors should be labeled as False.
  3. Warning fix (e.g., pylint warning) are not considered to a bug fix and should be labeled as False.
  4. Unless the change is aimed to solve a specific known problem, it is not a fix. For example, in “improving reliability”, one cannot point at the problem being fixed. These cases should be labeled as False.
  5. In cases that the commit contains some modifications, if at least one of them is considered a bug fix , the commit should be labeled as True.
  6. Fix to build/compile, are considered a bug fix and should be labeled as True.
  7. Tests are considered to be a part of the system and its requirements. Therefore, a bug fix in the tests is a bug fix and should be labeled as True.
