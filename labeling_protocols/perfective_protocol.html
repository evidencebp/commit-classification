1. Any refactor is perfective
  2. Perfective but not refactor
    1. Improving style is perfective yet not a refactor.
    2. Clean up not in code (e.g., documentation) is perfective but not a refactor
    3. Fix documentation, adding documentation - perfective but not refactor.
    4. Indentation, code style changes are perfective but not refactor
  3. Upgrading a library, is adaptive, not perfective is functionally is changed
  4. Bumping self version is perfective, not adaptive
  5. Gitignore actions are perfective
  6. When conventional commits appear, we base the decision upon them. The labels 'refactor', 'docs', and 'style' are perfective.
