We classify text for security related modifications.
Instructions
  1. We consider a commit as a security related commit from the point of view of a malicious actor. Hence a commit is security related even of the developer does not consider it to be so.
  2. For example, passwords are important for security and in general any work related to them is security related.
  3. Though network is a common attack vector, it is too wide so networking alone is not considered to be security related.
  4. When labeling for security fixes we label for 1. Including a fix 2. Related to security 3. Including a security related fix. Tangled commits might be related to security, have a bug fix and no security related fix.
